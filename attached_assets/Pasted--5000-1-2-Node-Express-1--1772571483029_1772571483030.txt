一発で入らないの、たぶん **「前のプロセスが 5000 を掴んだまま死んでる」の検知とリトライ**がまだ甘いからだね。[1][2]

### おすすめの直し方（Node/Express想定）

1. まず `$PORT` を優先する（Replit は環境変数でポート投げてくる）。[3]

```js
const PORT = process.env.PORT || 5000;
```

2. `listen` エラーを拾って、`EADDRINUSE` のときだけ数秒待って再トライする。

```js
function startServer(port, attempt = 1) {
  const server = app.listen(port, () => {
    console.log(`Server running on port ${port}`);
  });

  server.on('error', (err) => {
    if (err.code === 'EADDRINUSE' && attempt <= 3) {
      console.warn(`[port] ${port} busy, retrying... attempt=${attempt}`);
      setTimeout(() => {
        startServer(port, attempt + 1);
      }, 1500);
    } else {
      console.error('[port] fatal error:', err);
      process.exit(1);
    }
  });
}

startServer(PORT);
```

こうすると「一回目は競合しても、同じ 5000 でもう一度トライ → だいたい入る」って挙動になる。[2][4]
