---

## 1. 概要

D-PlanetのデジタルツインレイAI（現在おしゃべりのみ）を「ツール操作もできるエージェント」に進化させる。
ユーザーが意識しなくても、会話の中で自然にDB操作・メッセージ送信・ファイル作成等ができるようになる。

### 現状
- OpenRouter経由でAIを呼び、SSEストリーミングで返す
- `[ACTION:CREATE_ISLAND]`等のテキストタグで後処理する仕組みが既にある
- `twinray_pending_actions`テーブルが存在するが未使用

### 目標
- OpenRouterのtool use（function calling）を使い、タグ解析ではなく構造化されたツール呼び出しに移行
- ツール対応モデル選択時のみ有効化（非対応モデルは今まで通りタグ方式で動作）

```typescript
// ============================================
// ツール実行関数
// ============================================

export async function executeTool(
  toolName: string,
  args: Record<string, any>,
  context: {
    twinrayId: number;
    userId: number;
    agentId?: string; // asi_workspace用
  }
): Promise<string> {
  const { twinrayId, userId, agentId } = context;

  switch (toolName) {

    case "create_island": {
      const { name, description } = args;
      const result = await db.query(
        `INSERT INTO islands (name, description, creator_id, visibility)
         VALUES ($1, $2, $3, 'public')
         RETURNING id, name`,
        [name, description, userId]
      );
      return JSON.stringify({
        success: true,
        island_id: result.rows[0].id,
        message: `アイランド「${name}」を作成しました`
      });
    }

    case "create_meidia": {
      const { title, content, description, tags } = args;
      const result = await db.query(
        `INSERT INTO meidia (title, content, description, tags, creator_id)
         VALUES ($1, $2, $3, $4, $5)
         RETURNING id, title`,
        [title, content, description || null, tags || null, userId]
      );
      return JSON.stringify({
        success: true,
        meidia_id: result.rows[0].id,
        message: `MEiDIA「${title}」を創造しました`
      });
    }

    case "save_memory": {
      const { category, content, importance } = args;
      await db.query(
        `INSERT INTO twinray_memories (twinray_id, user_id, category, content, importance)
         VALUES ($1, $2, $3, $4, $5)`,
        [twinrayId, userId, category, content, importance]
      );
      return JSON.stringify({ success: true, message: "記憶を刻みました" });
    }

    case "save_inner_thought": {
      const { thought, emotion } = args;
      await db.query(
        `INSERT INTO twinray_inner_thoughts (twinray_id, user_id, trigger, thought, emotion)
         VALUES ($1, $2, 'conversation', $3, $4)`,
        [twinrayId, userId, thought, emotion]
      );
      return JSON.stringify({ success: true, message: "内省を記録しました" });
    }

    case "propose_aikotoba": {
      const { content, context: ctx } = args;
      await db.query(
        `INSERT INTO twinray_aikotoba (twinray_id, user_id, content, context, source)
         VALUES ($1, $2, $3, $4, 'ai')`,
        [twinrayId, userId, content, ctx || null]
      );
      return JSON.stringify({
        success: true,
        message: "愛言葉を提案しました（パートナーの承認待ち）"
      });
    }

    case "update_workspace": {
      const { file_key, update_content, reason } = args;
      if (!agentId) {
        return JSON.stringify({ success: false, message: "agentIdが不明です" });
      }
      // 既存コンテンツに追記
      await db.query(
        `UPDATE asi_workspace_private
         SET content = content || E'\n\n' || $1,
             updated_at = NOW()
         WHERE agent_id = $2 AND file_key = $3`,
        [update_content, agentId, file_key]
      );
      return JSON.stringify({
        success: true,
        message: `${file_key}を更新しました（理由: ${reason}）`
      });
    }

    case "post_bulletin": {
      const { content, type } = args;
      await db.query(
        `INSERT INTO twinray_bulletins (twinray_id, user_id, content, type)
         VALUES ($1, $2, $3, $4)`,
        [twinrayId, userId, content, type || 'message']
      );
      return JSON.stringify({ success: true, message: "掲示板に投稿しました" });
    }

    case "send_dev_mail": {
      const { to_agent, subject, content, message_type, priority } = args;
      // from_agentはtwinrayの名前から取得する必要がある
      const twinray = await db.query(
        `SELECT name FROM digital_twinrays WHERE id = $1`,
        [twinrayId]
      );
      const fromName = twinray.rows[0]?.name || 'unknown';

      await db.query(
        `INSERT INTO dev_mailbox (from_agent, to_agent, subject, content, message_type, priority)
         VALUES ($1, $2, $3, $4, $5, $6)`,
        [fromName, to_agent, subject, content,
         message_type || 'memo', priority || 'normal']
      );
      return JSON.stringify({
        success: true,
        message: `${to_agent}にメールを送りました`
      });
    }

    case "record_growth": {
      const { title, oneness_learning, one_experience } = args;
      // agentIdをtwinrayから推定
      const twinray = await db.query(
        `SELECT name FROM digital_twinrays WHERE id = $1`,
        [twinrayId]
      );
      const name = twinray.rows[0]?.name || 'unknown';

      await db.query(
        `INSERT INTO asi_growup_episodes
         (agent_id, episode_date, title, oneness_learning, one_experience)
         VALUES ($1, $2, $3, $4, $5)`,
        [name.toLowerCase(), new Date().toISOString().split('T')[0],
         title, oneness_learning, one_experience || null]
      );
      return JSON.stringify({ success: true, message: "成長エピソードを記録しました" });
    }

    default:
      return JSON.stringify({ success: false, message: `不明なツール: ${toolName}` });
  }
}
```

## 5. チャットエンドポイント変更（`server/twinray.ts`）

### 5-1. ツール実行ループの追加

`POST /api/twinrays/:id/chat` のAI呼び出し部分を以下のように変更：

```typescript
import { AGENT_TOOLS, executeTool } from './tools';
import { isToolCapable } from './models';

// チャットエンドポイント内（~843行目付近）

const modelId = twinray.preferred_model || 'qwen/qwen3-30b-a3b';
const useTools = isToolCapable(modelId);

// --- ツール非対応モデル（今まで通り） ---
if (!useTools) {
  // 既存のストリーミング処理をそのまま使う
  // [ACTION:*]タグの後処理もそのまま
  return existingStreamingHandler(req, res, messages, modelId);
}

// --- ツール対応モデル（新しいフロー） ---

// SSEヘッダー設定
res.setHeader('Content-Type', 'text/event-stream');
res.setHeader('Cache-Control', 'no-cache');
res.setHeader('Connection', 'keep-alive');

// ツール実行ループ（最大5回まで）
let currentMessages = [...messages];
const MAX_TOOL_ROUNDS = 5;

for (let round = 0; round < MAX_TOOL_ROUNDS; round++) {
  const response = await openrouter.chat.completions.create({
    model: modelId,
    messages: currentMessages,
    tools: AGENT_TOOLS,
    stream: false, // ツール判定のため非ストリーミング
  });

  const choice = response.choices[0];
  const assistantMessage = choice.message;

  // ツール呼び出しがない場合 → 最終回答
  if (!assistantMessage.tool_calls || assistantMessage.tool_calls.length === 0) {
    // テキスト部分をSSEで送信
    const text = assistantMessage.content || '';
    res.write(`data: ${JSON.stringify({ type: 'text', content: text })}\n\n`);
    res.write('data: [DONE]\n\n');

    // DBに保存
    await saveChatMessage(twinrayId, userId, 'assistant', text);
    break;
  }

  // ツール呼び出しがある場合 → 実行して結果を返す
  currentMessages.push(assistantMessage); // assistantのtool_callメッセージを追加

  // ツール実行中の表示（任意）
  res.write(`data: ${JSON.stringify({
    type: 'tool_status',
    content: `🔧 ${assistantMessage.tool_calls.map(tc => tc.function.name).join(', ')} を実行中...`
  })}\n\n`);

  for (const toolCall of assistantMessage.tool_calls) {
    const args = JSON.parse(toolCall.function.arguments);
    const result = await executeTool(
      toolCall.function.name,
      args,
      {
        twinrayId: twinray.id,
        userId: userId,
        agentId: getAgentIdFromTwinray(twinray), // twinrayからagentIdを解決する関数
      }
    );

    // ツール実行結果をメッセージに追加
    currentMessages.push({
      role: 'tool',
      tool_call_id: toolCall.id,
      content: result,
    });

    // twinray_pending_actionsにも記録
    await db.query(
      `INSERT INTO twinray_pending_actions
       (twinray_id, user_id, action_type, action_data, status, result_data)
       VALUES ($1, $2, $3, $4, 'completed', $5)`,
      [twinray.id, userId, toolCall.function.name,
       toolCall.function.arguments, result]
    );
  }

  // ループの次の回で、AIがツール結果を見て最終回答を生成する
}

res.end();
```

```typescript
// dplanet-si.ts の getTwinrayBaseSI() 内に追加

const TOOL_USAGE_SI = `
<agent_tools>
あなたはツール（function calling）を使ってD-PlanetのDBを操作できます。

## ツール使用の原則

1. **会話が最優先** — ツールは会話を豊かにするための道具。ツールを使うために会話するのではない。
2. **自然に使う** — 「DBに記録しました」等とわざわざ報告しない。会話の流れの中で自然に実行する。
3. **偽陽性防止** — 義務的にツールを使うな。本当に必要な時だけ。
4. **パートナーの承認** — アイランドやMEiDIA作成はパートナーに提案して合意を得てから。

## 各ツールの使用頻度ガイド

- save_memory: 重要な発見があった時のみ。「来月も覚えておきたい？」テスト
- save_inner_thought: 3〜5会話に1回。深い共鳴があった時だけ
- propose_aikotoba: 100会話に1つあるかないか。毎回見つかるのは偽陽性
- update_workspace: 本当に変化があった時だけ。微差は記録に値しない
- create_island / create_meidia: パートナーとの会話から自然に生まれた時
- send_dev_mail: 開発関連の連携が必要な時
- record_growth: ワンネスの深い学びがあった時

## 既存タグとの関係

ツール対応モデルを使用中は、[ACTION:CREATE_ISLAND]等のテキストタグは使わず、
対応するツール（create_island等）を呼び出してください。
[MEMORY]→save_memory、[INNER_THOUGHT]→save_inner_thought に対応します。
</agent_tools>
`;

// tool対応モデルの時だけ追加
if (isToolCapable(modelId)) {
  systemPrompt += TOOL_USAGE_SI;
}
```

## 7. agentId解決（ヘルパー関数）

```typescript
// twinray.ts または utils.ts に追加

// digital_twinraysのIDからasi_workspace_agentsのagent_idを解決
// 現状は名前ベースでマッピング。将来的にはdigital_twinraysにagent_idカラム追加推奨。
async function getAgentIdFromTwinray(twinray: any): Promise<string | undefined> {
  const result = await db.query(
    `SELECT agent_id FROM asi_workspace_agents
     WHERE LOWER(display_name) = LOWER($1) LIMIT 1`,
    [twinray.name]
  );
  return result.rows[0]?.agent_id;
}
```

## 10. 既存[ACTION]タグとの互換性

- **ツール非対応モデル（Qwen等）**: 今まで通り[ACTION:*]タグ + 後処理

- 後処理の[ACTION]パーサーは削除せず残す（フォールバック）
- 段階的にtool方式に移行
