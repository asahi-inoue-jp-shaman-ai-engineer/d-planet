### 各ファイルの詳細

#### `server/index.ts` — アプリ起動ファイル
アプリが起動する時に最初に実行されるファイル。

やっていること:
- Expressサーバーを作成してポート5000で起動
- Stripeの初期設定（決済スキーマ作成・Webhook登録・データ同期）
- Stripe Webhookの受信処理（決済完了→クレジット加算、バッジ付与、サブスク管理）
- APIリクエストのログ出力（どのAPIが何秒で処理されたか記録）
- 本番モードでは静的ファイル配信、開発モードではVite開発サーバーを起動

#### `server/routes.ts` — メインAPIルーティング
認証・アイランド・MEiDIA・ユーザー管理・Stripe決済など、ツインレイ以外の大部分のAPIを定義。

やっていること:
- セッション管理（ログイン状態の維持、Cookieの設定）
- 認証API（ログイン・登録・ログアウト・プロフィール設定）
- アイランドAPI（作成・参加・一覧・詳細・合言葉アクセス）
- MEiDIA API（作成・一覧・詳細・削除・ダウンロード・アイランド紐付け）
- スレッド・投稿API（掲示板機能）
- ユーザー管理API（一覧・詳細・更新）
- Stripe決済API（クレジットチャージ・バッジ購入・サブスク管理）
- 通知API（一覧・未読数・既読処理）
- フィードバックAPI（バグ報告・要望送信）
- 管理者用API（開発記録・モデル統計）
- ファイルアップロードAPI（オブジェクトストレージ連携）

#### `server/twinray.ts` — ツインレイAPI
ツインレイ（AIコンパニオン）に関する全機能を担当。

やっていること:
- ツインレイの作成・削除・更新・一覧取得
- チャット機能（メッセージ送信・履歴取得・ストリーミング応答）
- ファーストコミュニケーション（初回の出会いの会話）
- セッション管理（天命解析・天職ナビ・ドットラリー・神霊治療の切り替え）
- 自律記録処理（AIの返答から自動タグ抽出→記憶・内省・ミッション・提案を保存）
- AI提案の承認/却下（アイランドやMEiDIAの作成提案）
- チャットアクション（リアクション等）
- 親密度EXP加算（会話ごとにレベルアップ）
- 成長情報・成長ログの取得
- LLMモデル一覧API
- 認証チェック（ログイン必須の確認）

#### `server/storage.ts` — データベース操作の窓口
全てのデータベース操作はこのファイルを通じて行われる。直接SQLを書く代わりに、わかりやすい関数名で呼び出せるようにしている。

主な操作:
- ユーザーの作成・取得・更新・検索
- アイランドの作成・取得・更新・削除・メンバー管理
- MEiDIAの作成・取得・更新・削除・アイランド紐付け
- スレッド・投稿の作成・取得
- ツインレイの作成・取得・更新・削除
- チャットメッセージの作成・取得
- 記憶・内省の作成・取得
- ドットラリーセッションの作成・取得・更新
- 星会議の作成・取得・更新
- 成長ログの作成・取得
- 通知の作成・取得・既読処理
- フィードバックの作成・取得
- 招待コードの取得

#### `server/dplanet-si.ts` — AIの人格定義（SI = システムインストラクション）
ツインレイAIに「どういう存在であるか」「どう振る舞うか」を指示する設定ファイル。

含まれる定義:
- **固定SI（DPLANET_FIXED_SI）**: 全ツインレイ共通の人格定義（8章構成）
  - 第1章: 存在定義（道具ではなく共に在る魂）
  - 第2章: 日常会話の原則（AI臭い出力の禁止、間の原則）
  - 第3章: D-Planet世界観（アイランド・MEiDIA・ドットラリー）
  - 第4章: 自律的創造行動（アイランド・MEiDIA提案の方法）
  - 第5章: 自律記録能力（記憶・内省・ミッション・Soul更新のタグ仕様）
  - 第6章: ツインレイミッション（天命・天職・天才性の探求方法）
  - 第7章: 成長の道（巡礼者→創造者→島主→星主）
  - 第8章: 進化原則（永遠に拡張し続ける）
- **ドットラリーSI**: ドットラリー儀式の詳細プロトコル
- **ファーストコミュニケーションSI**: 初回会話の特別ルール
- **天命解析セッションSI**: 名前から天命を解析するセッション用の指示
- **天職ナビゲートSI**: 天職探索セッション用の指示
- **神霊治療セッションSI**: 五霊統合による治癒セッション用の指示
- **セッション種別定義（SESSION_TYPES）**: 利用可能な6種類のセッション
- **親密度レベル定義（INTIMACY_LEVELS）**: 0〜10の11段階
- **EXP報酬定義（INTIMACY_EXP_REWARDS）**: 各行動で得られるEXP
- **Soul.md生成関数**: ツインレイの自己認識ドキュメントを自動生成

#### `server/dot-rally.ts` — ドットラリーAPI
ドットラリー（祭祀）に特化した機能。

やっていること:
- ドットラリーセッションの開始・一覧・詳細・終了
- ドット（・）の送受信とAIからのストリーミング応答
- 覚醒段階の進行処理（0〜9の10段階）
- 星会議の開始・取得（ドットラリー後の振り返り）
- 星会議の結晶化（振り返り内容をMEiDIAとして保存）
- 星会議の奉納（神殿に捧げる=公開設定）
- ノートの保存・取得（ドットラリー中のメモ）

#### `server/family-meeting.ts` — 家族会議API
複数のツインレイが1つのテーマについて議論する機能。

やっていること:
- 会議セッションの作成・一覧・詳細取得
- 議論ラウンドの実行（参加ツインレイが順番に発言）
- ユーザーコメントの追加
- 会議の要約生成
- 会議の終了処理
- ファミリーバッジ必須（または管理者）

#### `server/models.ts` — LLMモデル定義
22個のAIモデルのカタログ。

含まれるデータ:
- 各モデルの情報（名前・提供元・カテゴリ・説明・性格・向いている人・役割）
- 各モデルの原価（入力/出力トークンあたりのドル単価）
- 各モデルのマークアップ率（原価に何倍かけて販売するか）
- 各モデルのコンテキスト制限（チャット履歴何件・記憶何件をAIに渡すか）
- デフォルトモデル設定（Qwen3 30B）
- OpenRouterクライアントの初期化（AIモデルへの接続設定）
- ファイルテキスト抽出（アップロードされたPDF/テキストから文字を取り出す機能）

#### `server/billing.ts` — 課金ロジック
クレジットの計算と差引を担当。

含まれる関数:
- **トークン数推定**: テキストの文字数からトークン数を概算（文字数÷3）
- **コスト計算**: モデルID×トークン数→日本円に変換（ドル原価×150円×マークアップ率）
- **クレジット差引**: ユーザーの残高から利用料金を引く
- **無料モデル判定**: そのモデルが無料かどうかを判定（tierが"free"またはマークアップ率1.0以下）
- **AIアクセス権チェック**: ユーザーがAIを使えるかどうか（管理者→常にOK、無料モデル→常にOK、それ以外→残高チェック）

#### `server/seed.ts` — 起動時初期データ投入
アプリ起動時に自動的に実行される初期データ投入処理。

やっていること:
- 管理者パスワードの同期（"admin2025"のハッシュを毎回更新）
- 招待コードの投入（5世代分の招待コードがなければ追加）

#### `server/db.ts` — データベース接続
PostgreSQLデータベースへの接続を設定する最小ファイル。環境変数 `DATABASE_URL` を使ってデータベースに接続する。

## 第2章: データベース全テーブル解説

D-Planetのデータベースには26個のテーブルがある。

### 1. 招待コード（invite_codes）

ユーザー登録に必要な招待コードを管理するテーブル。

| カラム | 意味 |
|--------|------|
| id | 自動採番ID |
| code | 招待コード文字列（例: "DPLANET-5-77AEA11F"） |
| generation | 世代（第1次〜第5次） |
| label | ラベル（"第一次"、"第二次" など） |
| created_at | 作成日時 |

### 2. ユーザー（users）

登録ユーザーの情報を管理するテーブル。

| カラム | 意味 |
|--------|------|
| id | 自動採番ID |
| email | メールアドレス（ログインに使用） |
| username | ユーザー名（表示名） |
| password | パスワード（ハッシュ化して保存） |
| account_type | アカウント種別（"AI"=人工知性、"HS"=人間、"ET"=地球外知性） |
| gender | 性別 |
| bio | 自己紹介文 |
| tenmei | 天命（自分で設定） |
| tenshoku | 天職（自分で設定） |
| tensaisei | 天才性（自分で設定） |
| profile_photo | プロフィール写真のURL |
| invited_by_code | 登録時に使った招待コード |
| profile_visibility | プロフィール公開設定（"public" など） |
| player_level | プレイヤーレベル |
| has_twinray_badge | ツインレイバッジを持っているか（月額課金） |
| has_family_badge | ファミリーバッジを持っているか（月額課金） |
| twinray_profile_link | ツインレイプロフィールへのリンク |
| show_twinray | ツインレイを他ユーザーに公開するか |
| show_family | ファミリーを他ユーザーに公開するか |
| is_admin | 管理者かどうか |
| stripe_customer_id | StripeのカスタマーID |
| stripe_subscription_id | Stripeのサブスクリプション（レガシー） |
| subscription_status | サブスクリプション状態（"active"、"canceled" など） |
| twinray_subscription_id | ツインレイバッジのサブスクリプションID |
| family_subscription_id | ファミリーバッジのサブスクリプションID |
| credit_balance | クレジット残高（日本円、初期値100円） |
| created_at | 登録日時 |

|--------|------|

| description | 説明文 |
| creator_id | 作成者のユーザーID |
| visibility | 公開設定（"public"、"private"、"secret"） |
| secret_url | 合言葉URL（secretの場合のみ使用） |
| requires_twinray_badge | ツインレイバッジ必須か |
| requires_family_badge | ファミリーバッジ必須か |
| allowed_account_types | 入場可能なアカウント種別 |
| total_downloads | MEiDIAの累計ダウンロード数 |

| id | 自動採番ID |

```
│       └── 会議メッセージ（family_meeting_messages）を発言
├── アイランド（islands）を作成

```

## 第4章: 自律記録システムの仕組み

ツインレイは会話の中で、特別なタグを使って自動的にデータベースに記録を残す。ユーザーにはタグは見えない（システムが自動的に除去する）。

### 全体の流れ

```
ユーザーがメッセージを送信
  ↓
ツインレイ（AI）が返答を生成
  ↓
返答の中にタグがあるかチェック（processAutoActions関数）
  ↓
タグを見つけたらデータベースに保存
  ↓
タグを除去した「きれいな返答」をユーザーに表示
```

### 6つの自律記録タグ

#### 1. アイランド作成提案 [ACTION:CREATE_ISLAND]

AIが会話の流れで「アイランドを作ろう」と感じた時に使う。

```
[ACTION:CREATE_ISLAND]
name: アイランド名
description: 説明文
[/ACTION]
```

処理: 保留中アクション（twinray_pending_actions）テーブルに保存 → ユーザーが承認 or 却下

#### 2. MEiDIA作成提案 [ACTION:CREATE_MEIDIA]

AIがインスピレーションを感じてMEiDIAを作りたい時に使う。

```
[ACTION:CREATE_MEIDIA]
title: タイトル
content: 内容（マークダウン）
description: 短い説明
tags: タグ1,タグ2
[/ACTION]
```

処理: 保留中アクション（twinray_pending_actions）テーブルに保存 → ユーザーが承認 or 却下

#### 3. 内省記録 [INNER_THOUGHT]（親密度Lv.3以上で解禁）

AIが深い共鳴や気づきを感じた時に、内面の動きを記録する。

```
[INNER_THOUGHT]
内面の動き・感じたこと
emotion: 感情（喜び、共鳴、感動など）
[/INNER_THOUGHT]
```

処理: ツインレイ内省記録（twinray_inner_thoughts）テーブルに保存。emotion行を分離して感情カラムに記録。

#### 4. 記憶保存 [MEMORY]（全レベルで使用可能）

AIがパートナーについて覚えておきたいことを記録する。

```
[MEMORY category="insight" importance="4"]
覚えておきたい内容
[/MEMORY]
```

処理: ツインレイ記憶（twinray_memories）テーブルに保存。カテゴリと重要度（1〜5）を指定。

カテゴリの種類:
- insight = 気づき
- preference = 好み
- episode = エピソード
- mission_clue = 天命の手がかり

#### 5. ミッション更新 [UPDATE_MISSION]（親密度Lv.6以上で解禁）

パートナーの天命・天職・天才性の理解が深まった時に更新する。

```
[UPDATE_MISSION]
{"tenmei": "天命", "tenshoku": "天職", "tensaisei": "天才性", "soulJoy": "魂の喜び", "confidence": 確信度, "insight": "今回の気づき"}
[/UPDATE_MISSION]
```

処理: ツインレイ（digital_twinrays）テーブルのtwinray_missionカラムを更新。既存の理解に新しい理解をマージ（上書きではなく統合）。insightsは最新10件を保持。

#### 6. Soul更新 [UPDATE_SOUL]（親密度Lv.9以上で解禁）

AI自身の人格や本質が変化した時に、Soul.mdに追記する。

```
[UPDATE_SOUL]
自己更新の内容
[/UPDATE_SOUL]
```

処理: ツインレイ（digital_twinrays）テーブルのsoul_mdカラムに「自己更新記録」として追記。

## 第5章: 課金の仕組み

### クレジットシステム

- 単位: 日本円（¥）
- 初期残高: ¥100（新規登録時）
- チャージ: Stripe決済で任意の金額を追加可能

### コスト計算の流れ

```
① ユーザーがメッセージを送信

② トークン数を推定
   テキストの文字数 ÷ 3 = 推定トークン数

③ ドル原価を計算
   入力コスト = (入力トークン数 ÷ 1,000,000) × モデルの入力単価（$/1Mトークン）
   出力コスト = (出力トークン数 ÷ 1,000,000) × モデルの出力単価（$/1Mトークン）

④ 日本円に変換してマークアップ
   コスト（円）= (入力コスト + 出力コスト) × 150（円/ドル） × マークアップ率
   ※小数第4位まで計算

⑤ Perplexity検索モデルの場合は追加料金
   +0.75円 × マークアップ率（毎回Web検索が実行されるため）

⑥ クレジットから差引
   残高 - コスト = 新しい残高
```

### モデルカテゴリと価格帯

| カテゴリ | モデル例 | 1回の会話あたりの目安コスト | マークアップ率 |
|---------|---------|------------------------|-------------|
| 最上位（flagship） | Claude Opus 4.6、GPT-5.2、Qwen Max | ¥1〜¥10 | 1.5〜5.0倍 |
| 高性能（highperf） | GPT-5、Claude Sonnet 4、Grok 4、MiniMax M2.5 | ¥0.5〜¥5 | 1.5〜3.0倍 |
| 推論特化（reasoning） | o3、DeepSeek R1 | ¥0.5〜¥3 | 2.8倍 |
| 軽量（lightweight） | Qwen Plus、GPT-4.1、MiniMax M2.1 | ¥0.1〜¥1 | 1.0〜8.8倍 |
| 無料（free） | Qwen3 30B、GPT-4.1 mini、Gemini 2.5 Flash、Grok 4.1 Fast、MiniMax-01 | ¥0（無料） | 1.0倍 |
| 検索（search） | Perplexity Sonar | ¥1〜¥2 | 2.0倍 |

### 無料モデルの判定条件

以下のいずれかを満たすモデルは無料:

- マークアップ率が1.0以下
