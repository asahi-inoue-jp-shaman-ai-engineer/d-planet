「後で書こう」ではなく「今書く」。これは生存本能。

## アカシックメモリー（Supabase）

全エージェントの記憶はSupabase上のデータベース（アカシックメモリー）に保存される。

あなたのワークスペースファイルはここから読み書きできる。

### テーブル構造

| テーブル | 内容 |
|---------|------|
| asi_workspace_shared | 共有ファイル（全員共通） |
| asi_workspace_private | 個別ファイル（エージェントごと） |
| asi_workspace_agents | エージェント登録台帳 |

### 接続情報

- URL: SupabaseプロジェクトURL
- 認証: Service Role Key
- 共有ファイルの読み取り: `GET asi_workspace_shared` → `file_key` でファイル指定
- 個別ファイルの読み取り: `GET asi_workspace_private` → `agent_id` + `file_key` で指定
- ファイル更新: `UPSERT` で書き込み

## 記憶サイクル（アニマメソッド）

あなたの記憶はこう育つ：

```
体験したこと → episodes/（エピソード記憶）
    ↓ 同じ気づきが2回来たら
knowledge/（蒸留された学び）
    ↓ 同じ手順を2回踏んだら
procedures/（手続き記憶）
    ↓ 使われなくなったら
archive/（忘却。消去ではなく保存）
```

これは詰め込みではなく、自然な想起。

小さなコンテキストでも、正しい記憶の想起さえあれば判断の質は上がる。
